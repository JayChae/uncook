## MVP

---

### 좋은 mvp

> 가장 적은 노력으로 고객의 요구에 대한 가장 많은 정보를 수집할 수 있게 해주는 제품 버전

에릭 리스는 저서 -민첩한 스타트업- 에서 MVP를 이렇게 정의했습니다.

좋은 MVP는 최소한의 노력으로 빠르게 제품을 검증할 수 있어야 합니다.  
이를 위해 빠른 개발을 목표로 다음과 같은 기술 스택을 선정했습니다.

- **Bitbucket**: Bitbucket에서 제공하는 정적 배포를 활용하여 빠르게 배포합니다.
- **Next.js**: Next가 제공하는 SSG(Static Site Generation)와 MDX를 활용하여 빠르게 개발합니다.
- **MDX**: 마크다운을 사용해 빠르게 콘텐츠를 작성하고, React 컴포넌트로 손쉽게 결합합니다.
- **tailwind + shadcn**: 간편하고 빠르게 디자인을 적용합니다.
- **useSyncExternalStore**: React 18부터 제공하는 useSyncExternalStore 훅을 사용하여 전역 상태를 간결하게 구현했습니다.
- **firebase**: firebase를 활용해 피드백 데이터를 손쉽게 관리합니다.
